# AI内容助手 V3：重新设计方案与 PRD

## 1. 设计方向调整

本版不再参考用户最初提供的工具类 App 截图，而是根据项目本身重新定位为：

> AI 内容分析与生成工作台

它不是简单的“工具合集”，而是帮助用户把短视频、直播、会议、培训等内容转化为可复用内容资产的 SaaS 产品。

---

## 2. 产品定位

### 2.1 一句话介绍

用户上传视频、直播录屏、会议录音或培训录音后，AI 自动完成转写、拆解、总结、复盘和内容生成，并输出短视频文案、直播计划、直播逐字稿、会议纪要、思维导图和 PPT 大纲。

### 2.2 产品核心价值

| 用户原始问题 | 产品解决方式 |
|---|---|
| 看不懂别人的爆款视频为什么火 | AI 拆解开头钩子、痛点、卖点、情绪点、转化点 |
| 没时间看完一整场直播 | AI 按时间轴整理直播内容框架 |
| 不知道下一场直播怎么讲 | AI 根据城市、账号定位和产品生成直播计划与逐字稿 |
| 会议/培训录音整理太麻烦 | AI 转写并生成纪要、行动项、导图、PPT |
| 内容多但没有沉淀 | 结果进入素材库，可搜索、复用、导出 |

- 大文件处理
- 批量任务
- 本地素材管理
- 复杂结果编辑

- 长直播录屏

- 团队内容资料沉淀

定位：

- 管理用户

## 6. C# 桌面客户端页面设计

- 批量导出。

## 10. 核心接口

### 创建任务

POST /api/tasks

```json
{
  "type": "short_video_analysis",
  "fileId": "F001",
  "title": "爆款视频拆解",
  "params": {
    "industry": "本地生活",
    "product": "团购套餐",
    "targetUser": "25-40岁城市白领",
    "outputType": "带货短视频文案"
  }
}
```

### 查询任务状态

GET /api/tasks/{taskId}

```json
{
  "taskId": "T001",
  "status": "processing",
  "progress": 68,
  "stage": "AI分析中"
}
```

### 获取任务结果

GET /api/tasks/{taskId}/result

```json
{
  "taskId": "T001",
  "status": "done",
  "result": {}
}
```

### 重新生成

POST /api/tasks/{taskId}/regenerate

```json
{
  "target": "copywriting",
  "instruction": "更口语化，适合抖音口播"
}
```

### 导出文件

POST /api/export

```json
{
  "taskId": "T001",
  "format": "ppt",
  "templateId": "PPT001"
}
```

## 12. 开发计划

| 阶段 | 周期 | 功能 | 验收标准 |
|---|---|---|---|
| 第1阶段 | 2-4周 | 登录、上传、任务、短视频拆解、文案生成 | 上传视频后输出核心点和文案 |
| 第2阶段 | 3-5周 | 直播录屏、时间轴、直播计划、逐字稿 | 上传直播后输出复盘报告 |
| 第3阶段 | 2-4周 | 会议录音、纪要、行动项、PPT大纲 | 上传录音后输出纪要和PPT |
| 第4阶段 | 4-6周 | C#客户端、批量任务、结果编辑器 | 客户端可批量处理和导出 |
| 第5阶段 | 2-4周 | 会员、订单、后台、成本看板 | 可完成商业化闭环 |

## 13. MVP 建议

第一版不要贪多，建议优先做：

1. 微信小程序。
2. 上传短视频。
3. AI 转写。
4. 短视频拆解。
5. 原创文案生成。
6. 历史任务。
7. 复制和导出。
8. 简单会员权益限制。

先验证用户是否愿意为“爆款拆解 + 原创文案”付费，再扩展直播和会议场景。
